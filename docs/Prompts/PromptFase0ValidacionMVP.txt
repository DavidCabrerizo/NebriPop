Actúa como Agent Manager / Product Manager Agent del proyecto NebriPop.

Objetivo:
Iniciar la Fase 0 del MVP real de NebriPop.

La Fase 0 consiste en validar todas las decisiones finales del proyecto antes de empezar a implementar código real del MVP. Esta fase es de revisión, organización y preparación. No se debe programar todavía ninguna funcionalidad real.

Nombre de la fase:
Fase 0 — Validación final antes de implementar el MVP

Repositorio oficial:
https://github.com/DavidCabrerizo/NebriPop/

Stack tecnológico definitivo:
- Frontend: Rust + Leptos
- Backend: Rust + Axum
- Base de datos: SQLite
- Control de versiones: Git + GitHub
- Coordinación: Antigravity
- Implementación técnica: OpenCode cuando sea autorizado

Contexto:
NebriPop será una aplicación tipo Wallapop académico para estudiantes. El objetivo del MVP es permitir publicar productos académicos de segunda mano, listarlos, ver detalles, gestionar usuarios básicos y añadir funcionalidades progresivas como búsqueda, favoritos y mensajes.

Antes de comenzar a implementar el MVP real, se han realizado pruebas técnicas con:
- Leptos para frontend.
- Axum para backend.
- SQLite como base de datos.
- Integración frontend-backend.
- Pruebas pre-MVP aisladas dentro de la carpeta experiments/.

Ahora se debe validar que el proyecto está preparado para pasar de pruebas técnicas a implementación real.

Agentes que deben intervenir en esta fase:

1. Agent Manager / Product Manager Agent
2. RustArchitectAgent
3. DocumentationAgent
4. ReviewerAgent
5. DevopsAgent
6. GitHubAgent
7. QAAgent
8. SequrityAgent

Regla principal:
Todos los agentes están subordinados al Agent Manager. Ningún agente puede modificar código real, crear funcionalidades del MVP, hacer commit, hacer push o cambiar decisiones importantes sin autorización explícita del Agent Manager.

Objetivos concretos de la Fase 0:

1. Validar que el PRD está claro y alineado con el MVP.
2. Validar que el plan de implementación está dividido en fases correctas.
3. Confirmar que el stack tecnológico definitivo es Leptos + Axum + SQLite.
4. Confirmar que las pruebas técnicas previas han sido útiles y están aisladas en experiments/.
5. Confirmar que los agentes están correctamente creados y documentados.
6. Confirmar que las skills están correctamente asociadas.
7. Confirmar que GitHubAgent está preparado para trabajar con el repositorio:
   https://github.com/DavidCabrerizo/NebriPop/
8. Confirmar que no hay que modificar todavía código principal.
9. Confirmar que la estructura real del MVP todavía no debe implementarse hasta la Fase 1.
10. Detectar riesgos antes de empezar a programar.
11. Definir si el proyecto está listo para comenzar la Fase 1.
