Contexto del proyecto:
Estamos desarrollando NebriPop, una aplicación tipo Wallapop académico para estudiantes. El proyecto utiliza Rust tanto en frontend como en backend, SQLite como base de datos y se desarrolla con una metodología asistida por IA usando Antigravity y OpenCode.

Queremos crear una skill específica que pueda utilizar posteriormente el agente DevopsAgent para encargarse de tareas relacionadas con DevOps, CI/CD, automatización, despliegue, configuración de entornos y documentación de ejecución del proyecto.

Tu tarea:
Crear una nueva skill llamada:

DevopsSkill

Además, guarda este prompt completo en el archivo:

docs/Prompts/PromptSkillCreatorDevops.txt

Si la carpeta docs/Prompts no existe, créala.

Nombre de la skill:
DevopsSkill

Skill base utilizada para crearla:
skill-creator

Objetivo principal de la skill:
La skill DevopsSkill debe servir para que un futuro DevopsAgent pueda planificar, revisar, implementar y documentar tareas relacionadas con DevOps en el proyecto NebriPop, especialmente:

- Preparación del entorno de desarrollo.
- Automatización de comandos.
- Configuración de scripts.
- Gestión de variables de entorno.
- Configuración de CI/CD.
- GitHub Actions.
- Build del backend Rust.
- Build del frontend Rust.
- Ejecución automática de tests.
- Validación de formato y calidad del código.
- Preparación de despliegue.
- Documentación de ejecución y despliegue.
- Revisión de riesgos de entorno.

Descripción de la skill:
DevopsSkill es una skill especializada en automatización, integración continua, despliegue y configuración de entornos para proyectos full-stack desarrollados con Rust. Su objetivo es ayudar a que NebriPop pueda ejecutarse, probarse y desplegarse de forma ordenada, reproducible y documentada.

La skill debe estar pensada para un proyecto académico, pero con buenas prácticas profesionales. Debe priorizar soluciones simples, mantenibles y fáciles de explicar en la memoria del proyecto.

Ámbito de aplicación:
La skill DevopsSkill se aplicará principalmente sobre:

- Repositorio del proyecto.
- Configuración de entorno.
- Archivos `.env.example`.
- Scripts de ejecución.
- GitHub Actions.
- Workflows de CI/CD.
- Comandos de build.
- Comandos de test.
- Documentación de despliegue.
- Docker, solo si está justificado.
- SQLite en entorno local.
- Backend Rust.
- Frontend Rust.

Jerarquía de uso:
Esta skill será utilizada por el futuro agente DevopsAgent.

El DevopsAgent estará subordinado al Agent Manager / Product Manager Agent, por lo que la skill debe reforzar estas reglas:

- No realizar tareas fuera del alcance aprobado.
- No modificar archivos críticos sin justificarlo.
- No cambiar la arquitectura sin autorización.
- No cambiar tecnologías principales del proyecto.
- No añadir complejidad innecesaria.
- Priorizar el MVP.
- Documentar todos los cambios importantes.
- Coordinarse con otros agentes cuando sea necesario.

Funciones principales que debe permitir la skill:

1. Preparación del entorno

La skill debe ayudar a definir:

- Requisitos previos del sistema.
- Instalación de Rust.
- Instalación de herramientas necesarias.
- Configuración de SQLite.
- Variables de entorno.
- Comandos para iniciar backend.
- Comandos para iniciar frontend.
- Comandos para ejecutar tests.
- Comandos para compilar el proyecto.

2. Automatización de comandos

La skill debe poder proponer o crear, cuando se autorice:

- Scripts de arranque.
- Scripts de build.
- Scripts de test.
- Scripts de limpieza.
- Scripts de inicialización de base de datos.
- Scripts de ejecución local.

Los scripts deben ser simples, claros y multiplataforma cuando sea posible.

3. CI/CD

La skill debe ayudar a definir y crear workflows de GitHub Actions para:

- Comprobar formato del código Rust.
- Ejecutar `cargo check`.
- Ejecutar `cargo test`.
- Compilar backend.
- Compilar frontend.
- Validar que no faltan archivos importantes.
- Ejecutar comprobaciones básicas antes de aceptar cambios.

4. Despliegue

La skill debe ayudar a planificar el despliegue del proyecto:

- Despliegue local.
- Despliegue académico sencillo.
- Posible despliegue en Render, Railway, Fly.io, Shuttle, VPS o similar si el Agent Manager lo autoriza.
- Documentación de variables de entorno.
- Estrategia de build.
- Estrategia de ejecución.
- Riesgos de despliegue.

5. Documentación DevOps

La skill debe ayudar a crear documentación como:

- `docs/DEPLOYMENT_GUIDE.md`
- `docs/DEVOPS_GUIDE.md`
- `docs/CI_CD.md`
- `docs/ENVIRONMENT_SETUP.md`
- `docs/LOCAL_RUN_GUIDE.md`
- `README.md`, solo en secciones relacionadas con ejecución y despliegue.

6. Gestión de variables de entorno

La skill debe revisar y proponer:

- `.env.example`
- Variables necesarias para backend.
- Ruta de SQLite.
- Puerto del backend.
- URL de API usada por el frontend.
- Secretos de autenticación.
- Buenas prácticas para no subir secretos reales al repositorio.

7. Seguridad básica DevOps

La skill debe evitar:

- Subir `.env` real.
- Exponer claves privadas.
- Exponer tokens.
- Hardcodear secretos.
- Publicar rutas sensibles.
- Dejar credenciales en workflows.

8. Coordinación con otros agentes

La skill debe estar diseñada para trabajar con:

- Agent Manager / Product Manager Agent.
- RustArchitectAgent.
- RustBackend.
- RustFrontend.
- DatabaseAgent.
- QAAgent.
- SequrityAgent.
- DocumentationAgent, si existe.

Reglas de coordinación:

- Si la tarea afecta a backend, coordinar con RustBackend.
- Si la tarea afecta a frontend, coordinar con RustFrontend.
- Si la tarea afecta a SQLite o migraciones, coordinar con DatabaseAgent.
- Si la tarea afecta a tests, coordinar con QAAgent.
- Si la tarea afecta a secretos o seguridad, coordinar con SequrityAgent.
- Si la tarea afecta a arquitectura, elevar al Agent Manager y coordinar con RustArchitectAgent.

Principios que debe aplicar la skill:

- Simplicidad.
- Reproducibilidad.
- Automatización razonable.
- Documentación clara.
- Separación de entornos.
- No subir secretos.
