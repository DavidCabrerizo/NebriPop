Actúa como Agent Manager / Product Manager Agent del proyecto NebriPop.

Objetivo:
Realizar una prueba controlada de código utilizando el framework frontend Rust Leptos antes de comenzar con la implementación real de NebriPop.

La prueba consistirá en crear una aplicación sencilla de calculadora básica con las siguientes operaciones:

- Suma
- Resta
- Multiplicación
- División

Esta prueba debe servir para validar:
- Que el equipo puede trabajar con Leptos.
- Que RustFrontend puede implementar una interfaz sencilla en Rust.
- Que RustArchitectAgent puede planificar una pequeña arquitectura frontend.
- Que QAAgent puede definir pruebas básicas.
- Que ReviewerAgent puede revisar el resultado.
- Que DocumentationAgent puede documentar la prueba.
- Que el flujo sigue estando controlado por Agent Manager.

Nombre de la prueba:
PruebaLeptosCalculadoraV1

Guarda este prompt completo en:

docs/Prompts/PruebaLeptosCalculadoraV1.txt

Si la carpeta docs/Prompts no existe, créala.

No modifiques docs/Logs.md directamente.

Una vez finalizada la prueba, prepara un resumen para que @DocumentationAgent registre el cambio en docs/Logs.md.

El resumen debe incluir:

- Fecha.
- Agente responsable.
- Acción realizada.
- Archivos creados o modificados.
- Motivo del cambio.
- Funcionalidades implementadas.
- Próximo paso recomendado.

Contexto técnico:
El proyecto NebriPop usará Rust tanto en frontend como en backend. Para el frontend se ha decidido probar Leptos como framework Rust para construir interfaces web reactivas.

Esta prueba no forma parte de la funcionalidad final de NebriPop. Es una prueba técnica aislada para validar el uso de Leptos.

Agentes que deben intervenir:

1. Agent Manager / Product Manager Agent
2. RustArchitectAgent
3. RustFrontend
4. QAAgent
5. ReviewerAgent
6. DocumentationAgent

Flujo obligatorio:

## 1. Agent Manager
Debe iniciar la prueba y definir claramente el alcance.

## 2. RustArchitectAgent
Debe planificar la arquitectura mínima de la calculadora con Leptos.

## 3. RustFrontend
Debe implementar la calculadora básica en Leptos siguiendo la planificación de RustArchitectAgent y las instrucciones del Agent Manager.

## 4. QAAgent
Debe definir una pequeña estrategia de prueba para la calculadora.

## 5. ReviewerAgent
Debe revisar el código creado.

## 6. DocumentationAgent
Debe documentar la prueba realizada.
