Actúa como Agent Manager / Product Manager Agent del proyecto NebriPop.

Objetivo:
Evaluar si los cambios actuales del proyecto son relevantes y, si lo son, activar automáticamente a @GitHubAgent para subirlos al repositorio oficial.

Repositorio oficial:
https://github.com/DavidCabrerizo/NebriPop/

Tarea:

1. Revisa el estado actual del proyecto.
2. Determina si existen cambios dignos de subir al repositorio.
3. Si los cambios son relevantes, autoriza a @GitHubAgent a:
   - revisar git status
   - revisar git diff
   - comprobar remoto
   - comprobar archivos sensibles
   - preparar commit
   - hacer commit
   - hacer push al repositorio oficial

Repositorio esperado:
https://github.com/DavidCabrerizo/NebriPop/

Instrucciones para @GitHubAgent:

Actúa como GitHubAgent usando la skill github-version-control.

El Agent Manager autoriza esta revisión y, si todo es correcto, la subida automática de cambios al repositorio oficial.

Debes:

1. Ejecutar revisión del repositorio:
   - git status
   - git remote -v
   - git branch
   - git diff
   - git diff --staged

2. Confirmar que el remoto apunta a:
   https://github.com/DavidCabrerizo/NebriPop/

3. Detectar archivos nuevos, modificados o eliminados.

4. Comprobar que no se suben archivos sensibles:
   - .env
   - .env.local
   - tokens
   - credenciales
   - claves privadas
   - target/
   - archivos temporales
   - bases de datos con datos sensibles

5. Si detectas riesgos, detén el proceso y avisa al Agent Manager.

6. Si no detectas riesgos, prepara un commit con mensaje profesional.

7. Ejecuta:
   - git add .
   - git commit -m "mensaje adecuado"
   - git push origin rama_actual

8. Después del push, devuelve:
   - rama usada
   - commit creado
   - archivos subidos
   - resumen de cambios
   - confirmación de push
   - resumen para @DocumentationAgent

No modifiques docs/Logs.md directamente.

Después del commit y push, prepara un resumen para @DocumentationAgent con:

- Fecha.
- Agente responsable: GitHubAgent.
- Acción realizada: commit y push al repositorio oficial.
- Rama utilizada.
- Commit creado.
- Archivos subidos.
- Motivo del commit.
- Resumen de cambios.
- Repositorio actualizado:
  https://github.com/DavidCabrerizo/NebriPop/
- Próximo paso recomendado.

Restricciones:

- No hagas force push.
- No borres ramas.
- No subas secretos.
- No subas .env.
- No subas target/.
- No cambies el remoto salvo autorización explícita.
- No modifiques código.
- Si hay conflicto o error, detente y avisa.

Al finalizar, Agent Manager debe devolver:

1. Si se han detectado cambios relevantes.
2. Si GitHubAgent ha podido subirlos.
3. Commit realizado.
4. Rama utilizada.
5. Estado final del repositorio.
6. Resumen para DocumentationAgent.
7. Próximo paso recomendado.
