Actúa como Agent Manager / Product Manager Agent del proyecto NebriPop.

Hemos validado manualmente que el backend Axum de la prueba de calculadora funciona correctamente.

Prueba realizada en PowerShell:

Invoke-RestMethod -Uri http://127.0.0.1:3000/operations `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"left_value":10.0,"operator":"+","right_value":5.0,"result":15.0}'

Resultado:
El backend guarda correctamente la operación y se puede consultar en:

http://127.0.0.1:3000/operations

Problema detectado:
Cuando realizamos operaciones desde el frontend Leptos, las operaciones no se guardan automáticamente en el backend.

Objetivo:
Corregir la integración del frontend Leptos para que cada operación realizada desde la calculadora se envíe correctamente al backend Axum y se actualice el historial.

Carpeta de trabajo:
experiments/fullstack_calculator_fusion/frontend/

Backend esperado:
http://127.0.0.1:3000

Endpoint para guardar operaciones:
POST http://127.0.0.1:3000/operations

Endpoint para consultar historial:
GET http://127.0.0.1:3000/operations

Instrucciones para RustFrontend:

Revisar el código del frontend Leptos y comprobar:

1. Que al pulsar suma, resta, multiplicación o división se calcula correctamente el resultado.
2. Que después de calcular el resultado se envía una petición POST a:

http://127.0.0.1:3000/operations

3. Que el body enviado tiene este formato exacto:

{
  "left_value": 10.0,
  "operator": "+",
  "right_value": 5.0,
  "result": 15.0
}

4. Que no se está usando una ruta relativa incorrecta como:

/operations

5. Que después de guardar la operación, el frontend hace una petición GET a:

http://127.0.0.1:3000/operations

6. Que si el backend no está activo, el frontend muestra un mensaje de error claro.

7. Que si se intenta dividir entre cero, no se envía la operación al backend y se muestra un error al usuario.

8. Que el historial se actualiza automáticamente después de cada operación guardada.
