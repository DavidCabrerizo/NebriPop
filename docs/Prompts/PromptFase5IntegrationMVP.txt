Actúa como Agent Manager / Product Manager Agent del proyecto NebriPop.

Objetivo:
Iniciar la Fase 5 del MVP real de NebriPop.

Nombre de la fase:
Fase 5 — Integración completa Frontend + Backend + SQLite

Repositorio oficial:
https://github.com/DavidCabrerizo/NebriPop/

Contexto:
Ya se han completado las fases anteriores del MVP de NebriPop:

- Fase 1: estructura real del proyecto.
- Fase 2: diseño de la base de datos SQLite.
- Fase 3: backend base con Axum conectado a SQLite.
- Fase 4: frontend base con Leptos conectado al backend.

Actualmente ya podemos visualizar la interfaz en navegador, pero necesitamos consolidar la integración completa para que todo el flujo funcione correctamente:

Leptos → Axum → SQLite → Axum → Leptos

NebriPop será una aplicación tipo Wallapop para compraventa de productos de segunda mano de todo tipo. En esta fase queremos asegurar que todos los productos, categorías y cambios realizados desde el frontend se guardan realmente en SQLite y no en archivos temporales, JSON ni memoria.

Stack tecnológico definitivo:
- Frontend: Rust + Leptos
- Backend: Rust + Axum
- Base de datos: SQLite
- Base de datos real esperada: database/nebripop.db
- Esquema SQL: database/schema.sql
- Datos iniciales: database/seeds.sql
- Backend local: http://127.0.0.1:3000
- Frontend local: http://127.0.0.1:8081
- Control de versiones: Git + GitHub

Objetivo principal de la Fase 5:
Validar y consolidar la integración completa entre frontend, backend y base de datos, asegurando que la app permite listar productos, ver detalles, crear productos y mantener los datos persistidos en SQLite.

Regla crítica de persistencia:
A partir de esta fase, cualquier cambio de datos realizado desde el backend debe guardarse en la base de datos SQLite `database/nebripop.db`.

Queda prohibido guardar productos, categorías, usuarios, favoritos, mensajes o cualquier dato del MVP en archivos `.json`.

No se deben usar archivos como:

- products.json
- categories.json
- users.json
- db.json
- data.json
- mock.json

Tampoco se debe usar almacenamiento temporal en memoria para datos principales del MVP, salvo casos estrictamente justificados como estados internos no persistentes.

Si algún agente detecta que el backend guarda o lee datos desde un `.json`, debe corregirlo para que use SQLite.

Base de datos:
Si `database/nebripop.db` no existe, debe generarse usando:

database/schema.sql
database/seeds.sql

La base de datos debe crearse dentro de:

database/nebripop.db

No se debe crear en otra carpeta salvo autorización explícita del Agent Manager.

Comandos esperados para generar la base de datos, si sqlite3 está disponible:

sqlite3 database/nebripop.db < database/schema.sql
sqlite3 database/nebripop.db < database/seeds.sql

Si sqlite3 no está instalado, DatabaseAgent debe documentar una alternativa o indicar cómo instalarlo, pero la fuente de verdad seguirá siendo `schema.sql` y `seeds.sql`.
