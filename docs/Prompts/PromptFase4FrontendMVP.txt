Nombre de la fase:
Fase 4 — Frontend base con Leptos conectado al backend Axum

Repositorio oficial:
https://github.com/DavidCabrerizo/NebriPop/

Contexto:
Ya se ha completado la Fase 3, donde se implementó el backend base real de NebriPop usando Rust + Axum conectado a SQLite. Ahora vamos a implementar el frontend base real usando Rust + Leptos, consumiendo los endpoints creados en la Fase 3.

NebriPop será una aplicación tipo Wallapop para compraventa de productos de segunda mano de todo tipo. En esta fase el objetivo es crear la primera interfaz visible del MVP, permitiendo listar productos, ver el detalle de un producto y publicar productos básicos desde el frontend.

Stack tecnológico definitivo:
- Frontend: Rust + Leptos
- Backend: Rust + Axum
- Base de datos: SQLite
- Backend URL local: http://127.0.0.1:3000
- Frontend URL local: http://127.0.0.1:8081
- Control de versiones: Git + GitHub
- Coordinación: Antigravity
- Implementación técnica: OpenCode cuando sea autorizado

Objetivo principal de la Fase 4:
Implementar el frontend base del MVP con Leptos, conectado al backend Axum, permitiendo visualizar productos, consultar categorías, crear productos y ver el detalle de un producto.

- Imagen principal si existe.

### Formulario para publicar producto

Debe incluir campos:

- category_id
- title
- description
- price
- condition
- location
- main_image_url, opcional

Debe enviar a:

POST http://127.0.0.1:3000/products

Body esperado:

{
  "user_id": 1,
  "category_id": 1,
  "title": "iPhone 12 usado",
  "description": "iPhone 12 en buen estado",
  "price": 250.0,
  "condition": "used",
  "location": "Madrid",
  "main_image_url": null
}

Notas:
- user_id puede enviarse temporalmente como 1 hasta que exista login funcional.
- Este comportamiento debe quedar documentado como solución temporal de MVP.
- category_id debe elegirse desde las categorías cargadas del backend.
- condition debe ser uno de: new, like_new, good, used, damaged.

Validaciones frontend obligatorias:

- title no puede estar vacío.
- description no puede estar vacía.
- price debe ser mayor o igual que 0.
- category_id debe estar seleccionado.
- condition debe estar seleccionada.
- location no puede estar vacía.
- main_image_url es opcional.
- Mostrar error si el backend devuelve error.
- Mostrar error si el backend no está encendido.

- Registro.
- JWT.
- Favoritos.
- Mensajes.
- Upload real de imágenes.
- Chat.
- Pagos.
- Notificaciones.
- Sistema real de compras.
- Búsqueda avanzada.
- Filtros avanzados.
