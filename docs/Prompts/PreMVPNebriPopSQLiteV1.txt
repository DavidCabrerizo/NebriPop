Funcionalidad de la mini app:
Crear una aplicación sencilla donde un estudiante pueda publicar un producto académico de segunda mano y verlo en un listado.

La app debe permitir:

1. Crear un producto.
2. Guardar el producto en SQLite.
3. Consultar productos guardados.
4. Mostrar los productos en el frontend.
5. Ver que los datos permanecen aunque se reinicie el backend.

Campos del producto:

- id
- title
- description
- price
- category
- condition
- location
- contact
- created_at

## 5. RustBackend

Debe implementar el backend con Rust + Axum.

Carpeta:

experiments/pre_mvp_nebripop_sqlite/backend/

El backend debe usar SQLite como persistencia real.

Tecnologías recomendadas:

- Rust
- Axum
- Tokio
- Serde
- SQLx o rusqlite
- tower-http para CORS

Preferencia:
Usar SQLx con SQLite si es sencillo.
Si SQLx complica demasiado la prueba, usar rusqlite.

El backend debe incluir:

- Servidor en http://127.0.0.1:3000
- CORS habilitado para http://127.0.0.1:8081
- Conexión a SQLite
- Endpoint GET /health
- Endpoint GET /products
- Endpoint POST /products
- Manejo de errores en JSON
- Validaciones básicas
- Código limpio usando clean-rust-skill si está disponible

Endpoints obligatorios:

GET /health

Respuesta esperada:

{
  "status": "ok",
  "message": "Pre-MVP NebriPop backend funcionando"

8. El producto aparece en pantalla.
9. Si se reinicia el backend, el producto sigue guardado en SQLite.
