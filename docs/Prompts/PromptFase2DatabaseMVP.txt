Actúa como Agent Manager / Product Manager Agent del proyecto NebriPop.

Objetivo:
Iniciar la Fase 2 del MVP real de NebriPop.

Nombre de la fase:
Fase 2 — Diseño de la base de datos SQLite real del MVP

Repositorio oficial:
https://github.com/DavidCabrerizo/NebriPop/

Contexto:
Ya se ha completado la Fase 1, donde se creó la estructura real del proyecto NebriPop. Ahora vamos a diseñar la base de datos SQLite real del MVP, que será utilizada posteriormente por el backend Axum.

NebriPop será una aplicación tipo Wallapop para compraventa de productos de segunda mano de todo tipo. Los usuarios podrán publicar productos, consultar productos publicados, ver detalles, gestionar usuarios básicos y preparar futuras funcionalidades como favoritos y mensajes.

Stack tecnológico definitivo:
- Frontend: Rust + Leptos
- Backend: Rust + Axum
- Base de datos: SQLite
- Control de versiones: Git + GitHub
- Coordinación: Antigravity
- Implementación técnica: OpenCode cuando sea autorizado

Objetivo principal de la Fase 2:
Diseñar y preparar la base de datos SQLite real del MVP, definiendo las tablas principales, sus campos, relaciones, restricciones, datos iniciales y documentación.

Importante:
En esta fase NO se debe implementar todavía:
- Backend funcional completo.
- Endpoints reales.
- Frontend funcional.
- Login completo.
- Sistema de favoritos completo.
- Sistema de mensajes completo.
- Subida real de imágenes.
- Chat.
- Pagos.
- Notificaciones.
- Sistema de compras reales.

La Fase 2 se centra únicamente en base de datos, esquema SQL, seeds y documentación.
