Nombre de la fase:
Fase 3 — Backend base con Axum conectado a SQLite

Repositorio oficial:
https://github.com/DavidCabrerizo/NebriPop/

Contexto:
Ya se ha completado la Fase 2, donde se diseñó la base de datos SQLite real del MVP de NebriPop. Ahora vamos a implementar el backend base real usando Rust + Axum y conectándolo con la base de datos SQLite diseñada en la fase anterior.

NebriPop será una aplicación tipo Wallapop para compraventa de productos de segunda mano de todo tipo. En esta fase el objetivo es crear la primera API backend funcional para consultar categorías, listar productos, ver el detalle de un producto y crear productos básicos.

Stack tecnológico definitivo:
- Frontend: Rust + Leptos
- Backend: Rust + Axum
- Base de datos: SQLite
- Acceso a datos: SQLx o rusqlite, según decida RustBackend con aprobación del Agent Manager
- Control de versiones: Git + GitHub
- Coordinación: Antigravity
- Implementación técnica: OpenCode cuando sea autorizado

Objetivo principal de la Fase 3:
Implementar el backend base del MVP con Axum, conectado a SQLite, dejando disponibles los primeros endpoints necesarios para productos y categorías.

Endpoints obligatorios:

### GET /health

Debe devolver:

  "status": "ok",
  "message": "NebriPop backend funcionando"
}

### GET /categories

Debe devolver la lista de categorías desde SQLite.

Respuesta esperada:

[
  {
    "id": 1,
    "name": "Electrónica",
    "created_at": "2026-05-24T10:00:00"
  }
]

### GET /products

Debe devolver la lista de productos desde SQLite.

Respuesta esperada:

[
  {
    "id": 1,
    "user_id": 1,
    "category_id": 1,
    "title": "iPhone 12 usado",
    "description": "iPhone 12 en buen estado",
    "price": 250.0,
    "condition": "used",
    "location": "Madrid",
    "status": "available",
    "main_image_url": null,
    "created_at": "2026-05-24T10:00:00",
    "updated_at": "2026-05-24T10:00:00"
  }
]

### GET /products/:id

Debe devolver el detalle de un producto concreto.

Si el producto no existe, debe devolver error JSON claro con código HTTP adecuado.

### POST /products

Debe permitir crear un producto básico.

Body esperado:

{
  "user_id": 1,
  "category_id": 1,
  "title": "iPhone 12 usado",
  "description": "iPhone 12 en buen estado",
  "price": 250.0,
  "condition": "used",
  "location": "Madrid",
  "main_image_url": null
}

Validaciones obligatorias:

- category_id debe existir.
- title no puede estar vacío.
- description no puede estar vacía.
- price debe ser mayor o igual que 0.
- condition debe ser uno de: new, like_new, good, used, damaged.
- location no puede estar vacía.
- status debe establecerse por defecto como available.
- Si user_id es null o no se usa todavía, debe estar justificado por la ausencia de login funcional.
- No aceptar campos malformados.
- Devolver errores claros en JSON.

Respuesta esperada:

{
  "id": 1,
  "user_id": 1,
  "category_id": 1,
  "title": "iPhone 12 usado",
  "description": "iPhone 12 en buen estado",
  "price": 250.0,
  "condition": "used",
  "location": "Madrid",
  "status": "available",
  "main_image_url": null,
  "created_at": "2026-05-24T10:00:00",
  "updated_at": "2026-05-24T10:00:00"
}
